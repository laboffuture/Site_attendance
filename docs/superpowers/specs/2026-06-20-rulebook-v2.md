# TRGBI Attendance — Rule Book v2 (canonical spec)

**Date:** 2026-06-20
**Status:** Source of truth for the v2 expansion. Captures the requirements gathered in session (incl. the shift/OT matrix and role-responsibility image of 2026-06-20). Each buildable piece gets its own plan under `docs/superpowers/plans/`.
**Builds on:** the shipped v1 (auth, org, employees, scan capture + location-lock, OT approval, dashboards/reports, requests, missed-clockout sweep) and the in-flight `feature/log-attendance` (scan-only, geofence enforcement, Log Attendance page).

## 1. Roles & hierarchy

Four tiers, top → bottom: **Management → HR → PM → Supervisor.**
(`super_admin` is being merged into Management — rename pending; `pm`/`supervisor` unchanged.)

**Supervisor sub-types** (from the role image):
- **Site Supervisor** — marks movement (on-site / off-site / travel) and the attendance of all workers at the site; submits the day with remarks.
- **MEP Supervisor** — for a given day/site, a worker's in/out must be marked by the **same mapped supervisor** (M/E/P trades mapped to their supervisor).

Workers are **records, not users** (no login).

## 2. Shift & overtime matrix (the OT engine)

Standard shift = clock window **including breaks**; paid work = window − break. OT = time worked **beyond** the standard window, with the break rule shown.

| Scenario | Day | Window | Standard | Beyond standard → OT |
|---|---|---|---|---|
| Day general | Weekday | **08:00–17:00** | 9h clock, **1h break** → 8h work | 17:00–20:00 → OT, **no break** |
| Day → overnight | Weekday | 08:00–17:00, extends to next-day **05:00** | as above | extension → OT, **incl 1h break** |
| Night general | Weekday | **20:00–05:00 (crosses midnight)** | 9h clock, **1h break** → 8h work | beyond 05:00 → OT, **+ additional 1h break** |
| Sunday general | Sunday | **08:00–14:00** | 6h, **no break** | beyond 14:00 → OT, **1h break** |

Rules:
- **OT = max(0, worked − standard)**, applying the per-segment break rule above; pending until approved (existing OT approval chain).
- **Cross-midnight:** night shifts and overnight-extending day shifts span two calendar dates — the attendance record must support this (see §10).
- Shift definition is **per site** (day window, night window, Sunday rule, break minutes), with the scenario auto-selected by the In-scan time.

## 3. Attendance capture (scan-only)

- **Face scan is the only way** to create/update attendance — no manual type-in (locked in v1.5). Two scan surfaces, both kept:
  - **Fixed kiosk** (`/station`, station-key device).
  - **Supervisor "Log Attendance"** (logged-in supervisor, picks the site) — the **primary** method.
- **Geofence-first:** when a geofenced site is picked, the page **verifies GPS up front** and only **unlocks Scan when the device is confirmed inside** the radius (configurable **100–200 m**). Outside / no-fix → blocked with a clear reason.
- **Exact GPS** captured on every scan.
- **Log Attendance page** shows a live **"logged today" list** (each scan: worker, In/Out, time) + a count, and a **request** shortcut — so the page is informative, not blank.

## 4. Daily attendance approval / regularization chain

Attendance moves through a governed lifecycle (this is what replaces ad-hoc manual entry):

1. **Scanned** — raw In/Out from face scans during the day.
2. **Submitted** — at day end the **Site Supervisor reviews, adds a remark per worker** (what work was done today), and **submits the day**.
3. **Recommended (regularized)** — the **PM verifies** all entries and **recommends regularization** (can flag/adjust corrections, incl. missed clock-outs).
4. **Approved** — **HR/Management approves** → attendance is final for payroll.

Corrections (missed clock-outs, wrong times) are resolved **inside this chain** (regularization), not by a free type-in grid.

## 5. Movement & travel time

- The Site Supervisor marks **on-site / off-site**; a worker's day can span **multiple sites**.
- **Travel between sites is its own activity** with start/end — shown **separately** (not work, not a gap). Example: Site A 09:00–12:00, **travel 12:00–13:00**, Site B 13:00 onward.
- **Inter-site move request:** Supervisor requests a worker shift to another site → **PM recommends → HR approves** (extends the existing `requests` subsystem; offload pattern).

## 6. Employee data additions

- **Pay at registration:** **daily wage** amount, and **food allowance** (whether applicable + amount). Feeds payroll-ready reports later.
- **Face enrolment for existing workers:** the 171 imported workers have **no face**. Add a **capture/replace-face** action on the worker record so a supervisor can enrol a face onto an already-registered worker (today only the new-enrol flow captures a face).

## 7. Segregated views

Clear, separate lists so nothing is buried:
- **Attendance list** — daily attendance, scoped/filterable.
- **Action list** — what needs the user's action (submit / recommend / approve / resolve flags).
- **OT list** — overtime, segregated by status (pending / approved / rejected).

## 8. Daily remarks

Per-worker remark captured at the supervisor's daily **submit** (§4 step 2): a short note of the work that worker did today. Logged daily, visible in regularization + reports.

## 9. Out of scope (for now)

- Full payroll *run* (wage × days + allowances → payslips) — v2 adds the inputs (pay fields) + approved hours; the actual payroll computation/export is a later phase.
- Public mobile app — the `mobile/` plan stands; web first.

## 10. Data-model implications

- **Attendance** record: key on the **shift's start date**; store full-timestamp In/Out that may **span midnight**; add `shiftType` (day/night/sunday), break minutes applied, and a `segments[]` array for multi-site/travel days (`{ type: work|travel, siteId, start, end }`). Add the **approval lifecycle**: `status` (scanned → submitted → recommended → approved), `submittedBy/At`, `recommendedBy/At`, `approvedBy/At`, and the per-worker daily `remark`.
- **ProjectSite**: add a **shift definition** (day window, night window, Sunday rule, break minutes) — extends the existing `standardStartTime/EndTime` + coordinates/geofence.
- **Worker**: add `dailyWage`, `foodAllowance { applicable, amount }`. Face capture/replace reuses the existing `faceEncoding`/`photoUrl`.
- **User (Supervisor)**: optional `supervisorType` (site|mep) + trade/worker mapping for the MEP rule.
